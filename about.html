Time to get more serious...

HTML and CSS

My way into the web started by learning how to use HTML and CSS to build simple static web pages. Separation between content and structure from presentational elements was the key that drove me around.
My favourite readings were Eric Meyer's blog, A List Apart's articles, Molly Holzschlag's blog, Jeffrey Zeldman's blog and all the articles talking about Web Standards and Web Design I could find on the web.
I was mainly focused on understanding how to give the right structure to web pages and how to modify their aspect using external style sheets.

Discovering the backend

One day, I happened to encounter Matt Mullenweg's website and know about the existence of Wordpress, the most famous (nowadays) CMS around the world and powering more than 25% of the Web.
I started studying PHP on-line and on books, while getting acquainted with Wordpress and its behaviours on codex.wordpress.ord and reading Smashing Wordpress and Smashing Wordpress Themes by Thord Daniel Hedengren.

The transition from markup and formatting languages (like HTML and CSS) to programming languages is really hard and I tested it on myself. Even if I was able to deploy a wordpress site very easily, work on the PHP code to modify the existent loop or to create new ones and modify every presentational aspect through child theme, I feeled like I was missing something very important.
I lacked the overall perspective on what I was doing. I wasn't able to create or imagine the logic behind that.

Bloc and Ruby on Rails

After numerous searching throughout the web, the eager to gain a better understanding of what coding is, led me to Bloc Inc., an online service that delivers courses to people desiring to learn software development, web development and mobile development.

I chose the Web Development Bootcamp, that teaches how to build robust, full stack real web applications using the Rails framework, after studying the essentials of object-oriented programming with Ruby. Prior to this, I had absolutely no clue what a MVC framework was and only a skin-deep knowledge about how MySQL database worked.

Luckily, Bloc provides you with the possibility to meet, one or more times per week, a "mentor" (an experienced developer often free-lancer or coming from organizations like Google, Uber, Heroku etc.). And my choice fell onto Charlie Gaines, a free-lance developer, after reading his description page on Bloc (not available any more), where he was describing himself as a self-taught web developer. My choice was influenced by the idea that a self-taught teacher could be a better help for someone like me.

At Bloc I learned how to build fully functional web applications from scratch, what RESTful programming is about, to code implementing gems to simplify my work, to read gems' wikis to find a way to get them work in specific environments, to make use of external services taking advantage of their APIs and much more. You can take a look to my github repositories to see what I'm talking about.

Aftermath of Bloc...

However, the most important thing I learned at Bloc is that coding is a great 'intellectual' adventure and a neverending creative challenge. And this is the aspect I like most in this kind of job.
This course impacted on me also as a person because along with technical skills, I learned a useful lesson: the possibility to stay in an "uncomfortable" zone, where you really do not understand what you are doing, and transform this negative feeling into a push forward.

Since the Bloc graduation, I've been working on my coding skills through the codewars platform (where I complete at least one programming challenge every day) and technical readings on books and online.

I can honestly say that I'm really passionate about coding and I'm always looking for ways to expand what I currently know.
